void Test_Receiver::connect_EC ( CORBA_Environment &  ACE_TRY_ENV = TAO_default_environment ())

Connect to the event channel using the values extracted.

Parameters
ACE_TRY_ENVCORBA env for propogating exceptions
Returns
true if some offers were found

This is where we connect the Push supplier on the request event channel, and register the fact that we shall be sending requests with the event type specified. Additionally, we connect a push consumer on to the data channel, and subscribe to the event types as defined in the offer we have selected. We also register a callback that shall handle the data coming in from the sensor

So far, we have not done anything that really needs undoing before aborting. From this point on, the supplier needs to be disconnected on error
