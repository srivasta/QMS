Detailed Description

Goals and Requirements:

  • The Logger should listen for service requests
  • The Logger should be able to log QMS events pushed on to to the CORBA event channel
  • One should be able to create multiple independent logs using the logger, each log being associated with a separate storage back-end, and an ability to filter the recorded events by event type and supplier id
  • One should be able to modify attributes for each log, including security policies, back-ends, and filters
  • The Logger must be a well behaved citizen in the QMS toolkit
  • The logger must be able to report performance metrics and other statistics
  • The logger should be a managed QMS component, and present a management interface that allows, at a minimum, start, stop, pause, and resume commands, in addition to providing the QMS Control and Command interfaces common to all citizens
