Implementation of the common elements of a one shot probe.

Parameters
argcCount of command line arguments.
argvList of command line arguments.
Returns
returns the exit status of the program.

Writing a one shot probe – that is, a class of probes where the probe has no state, and does not implement recurrring data publication. Instead, for each request, the requested data is provided, and the probe exits. Any periodic functionality is provided by the Probe Proxy built into the sensor itself.

This makes implementing the probe easy: we just need to perform the following tasks:

  • Stores the Command line information into variables.
  • Calls the get_data function these variables.

The variables required to hold host1, host2 passed from the Child to Probe.
