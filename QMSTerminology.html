Terminology in QMS

QMS defines a generally usable framework where several entities interoperate. Based on the nature of the interaction and the services offered up by the entity or the services desired by the entity, the entity behaves as a citizen, a client, a sensor or a combination of the above. It is important that you be aware of this terminology to ensure that you understand the roles of the entities.

  • QMS Citizens
    • An entity that operates in the QMS framework and
      • Provides a standard CORBA interface for command and control
      • Publishes capabilities with the QMS Trader
  • QMS Client
    • An entity that uses the services of QMS
  • Adapter Object (standard OO definition)
    • A standard C++ object that provides a user-friendly abstraction for complex or non-standard objects
  • QMS Core
    • Required elements of the QMS framework without which functionality would be seriously compromised
  • QMS Plugins
    • Add-on components to the QMS framework that enhance the services and functionality provided
  • QMS Sensors
    • QMS citizen that is a data source that deploys probes
  • QMS Probes
    • A lightweight instrumentation element visible only to its QMS Sensor
